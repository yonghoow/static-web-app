CIML002 Week 2 Lab 2

scenario

As a recently hired LAN technician, your network manager has asked you to demonstrate your ability to configure a small LAN. Your tasks include configuring initial settings on two switches using the Cisco IOS and configuring IP address parameters on host devices to provide end-to-end connectivity. You are to use two switches and two hosts/PCs on a cabled and powered network.

Instructions
Configure the devices to fulfill the requirements below.

Requirements
Use a console connection to access each switch.
Connect lightning cable from PC (RS 232) to switch (console). Use terminal to access switch from PC.

Name Class-A and Class-B switches.
Switch>enable
Switch#configure terminal
Switch(config)#hostname Class-A or hostname Class-B
Class-A(config)# or CLass-B(config)#

Use the xAw6k password for all lines.
CLass-A(config)# line console 0
CLass-A(config-line)# password xAw6k
Class-A(config-line)# login
Class-A(config-line)# end
Class-A#

Use the 6EBUp secret password.
Class-A> enable
Class-A# configure terminal
CLass-A(config)# enable secret 6EBUp
CLass-A(config)# exit
Class-A#

Securing VTY line access
Class-A> enable
Class-A# configure terminal
Class-A(config)# line vty 0 15
Class-A(config-line)# password xAw6k
Class-A(config-line)# login
Class-A(config-line)# end
Class-A#

Encrypt all clear text passwords.
Class-A> enable
Class-A# configure terminal
Class-A(config)# service password-encryption
Class-A(config)# exit
Class-A#

Configure an appropriate message-of-the-day (MOTD) banner.
Class-A# configure terminal
Class-A(config)# banner motd # Authorized access only. Violators will be prosecuted ... law.#

Configure addressing for all devices according to the Addressing Table.
Class-A# configure terminal
Class-A(config)# interface VLAN 1
Class-A(config-if)# ip address 10.10.10.100 255.255.255.0 or ip address 10.10.10.150 255.255.255.0
Class-A(config-if)# no shutdown
Class-A(config-if)#
Class-A(config-if)# exit

Save your configurations.
Class-A# copy running-config startup-config

Verify connectivity between all devices.
Click Student-1 PC and then click the Desktop tab.
Click IP Configuration. In the Addressing Table above, you can see that the IP address for PC1 is 10.10.10.4 and the subnet mask is 255.255.255.0. Enter this information for Student 1 PC in the IP Configuration window.
Repeat steps for Student-2 PC.

Click Student-1 PC. Close the IP Configuration window if it is still open. In the Desktop tab, click Command Prompt.
Type the ping command and the IP address for S1 and press Enter.
C:\> ping 10.10.10.100

Note: Click Check Results to see your progress. Click Reset Activity to generate a new set of requirements. If you click on this before you complete the activity, all configurations will be lost.
