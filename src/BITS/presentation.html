Preventive Control

  1. Private Web access

    2. Put private information regarding Equifax's susceptible version of web framework on private access website not indexed by search engine.

  2. Data Retention Policy

    3. Avoid storing customer's sensitive information beyond the time necessary to provide the service.

  3. Update communication policy

    4. The person in charge of patching the web portal did not receive the alert as he/she was not part of the email recipient list. Thus, the patching process was not proactive.

  4. Security Awareness Framework

    5. Request management to have company-wide cyber security management framework and ensure its adequate staffing and budget. Equifax can also share its latest security framework with other companies.

  5. Web Application Firewall (WAF)

    6. It protects web applications from SQL injections and other attacks. WAF could have prevented the exploitation of Equifax's vulnerability.

Preventive Control

  1. Data Loss Prevention (DLS) system

    2. It identifies, monitors and potentially averts unauthorized moves of sensitive data in and out of the network. DLS could have prevented the massive out flow of 14-16 Gb of data for 76 days undetected.

  2. Segmentation of systems

    3. Equifax had shared file systems, notes or configuration files from one system and can be accessed from other systems

  3. Encryption for Personal Identifiable Information (PII)

    4. None of the PII from the stolen data was encrypted, therefore compromising the confidentiality of the stolen PII. Encryption, truncation, masking or hashing of the critical PII is needed.

  4. Strong password

    5. one database has a password which matched the name of the database. An online portal had default username "admin" and password "admin" for an administrative account.

  5. Knowledge in use of vulnerability scanners

    6. Equifax IS team scanned the system exposed to the internet at root directory, not the subdirectory where vulnerabil;ity was listed. Better training is needed.

Defective Control

  1. Intruder Detection and Prevention System (IDS/IPS)

    2. Equifax's IDS/IPS was not able to detect malicious traffic leaving the system due to an expired SSL certificate from Nov 2016 to July 2017. A working Automated SSL certificate management tool is needed.

  2. Network logs forensics

    3. Consult Mandiant (now part of Google cloud) security team. Both Equifax and Mandiant security teams analyzed digital forensic data, they found attackers installed a backdoor in Equifax's servers.

Corrective Control

  1. Server Images

    2. One way to correct online servers infected by malware is restoring from offline server images, which remain untouched by the threat actors.
