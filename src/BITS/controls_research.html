18.1.2.1 Ensure 'Allow Input Personalization' is set to 'Disabled'

Problem:

Text that is collected includes all outgoing messages in Windows Mail, and MAPI enabled email clients, as well as URLs from the Internet Explorer browser history. The information that is stored includes word frequency and new words not already known to the handwriting recognition engines (for example, proper names and acronyms). Deleting email content or the browser history does not delete the stored personalization data. Ink entered through Input Panel is collected and stored.

Improvement:

Automatic learning stops and any stored data is deleted.

18.4.8.1 Ensure 'Enable insecure guest logons' is set to 'Disabled'

Problem:

Clients that allow insecure guest logons are vulnerable to a variety of man-in-the-middle attacks that can result in data loss, data corruption, and exposure to malware. Additionally, any data written to a file server using an insecure guest logon is potentially accessible to anyone on the network

Improvement:

If you disable this policy setting, the SMB client will reject insecure guest logons and prevent man-in-the-middle-attack

18.4.11.2 Ensure 'Prohibit installation and configuration of Network Bridge on your DNS domain network' is set to 'Enabled'

Problem:

A Network Bridge thus allows a computer that has connections to two different networks to share data between those networks. Allowing users to create a Network Bridge increases the risk and attack surface from the bridged network.

Improvement:

Users cannot create or configure a Network Bridge, reduce risk of attack to share data between different networks.

18.4.11.3 Ensure 'Prohibit use of Internet Connection Sharing on your DNS domain network' is set to 'Enabled'

Problem:

Non-administrators should not be able to turn on the Mobile Hotspot feature and open their Internet connectivity up to nearby mobile devices.

Improvement:

Mobile Hotspot cannot be enabled or configured by Administrators and non-Administrators alike

18.4.11.4 Ensure 'Require domain users to elevate when setting a network's location' is set to 'Enabled'

Problem:

Allowing regular users to set a network location increases the risk and attack surface

Improvement:

Domain users must elevate when setting a network's location. It reduces the risk and attack surface.

18.4.21.1 Ensure 'Minimize the number of simultaneous connections to the Internet or a Windows Domain' is set to 'Enabled'

Problem:

Preventing bridged network connections can help prevent a user unknowingly allowing traffic to route between internal and external networks, which risks exposure to sensitive internal data

Improvement:

While connected to an Ethernet connection, Windows won't allow use of a WLAN (automatically or manually) until Ethernet is disconnected. It prevents exposure of internal data to external networks.

18.4.21.2 Ensure 'Prohibit connection to non-domain networks when connected to domain authenticated network' is set to 'Enabled'

Problem:

The potential concern is that a user would unknowingly allow network traffic to flow between the insecure public network and the enterprise managed network.

Improvement:

When the computer is already connected to either a non-domain based network or a domain based network over Ethernet, and a user attempts to create a manual connection to an additional network in violation of this policy setting, the existing Ethernet connection is maintained and the manual connection attempt is blocked.

18.4.23.2.1 Ensure 'Allow Windows to automatically connect to suggested open hotspots, to networks shared by contacts, and to hotspots offering paid services' is set to 'Disabled'

Problem:

Automatically connecting to an open hotspot or network can introduce the system to a rogue network with malicious intent.

Improvement:

Connect to suggested open hotspots, Connect to networks shared by my contacts, and Enable paid services will each be turned off and users on the device will be prevented from enabling them.

18.8.3.1 Ensure 'Include command line in process creation events' is set to 'Disabled'

Problem:

When this policy setting is enabled, any user who has read access to the security events can read the command-line arguments for any successfully created process. Command-line arguments may contain sensitive or private information such as passwords or user data.

Improvement:

Sensitive information like password and user data not shown in security events.

18.8.12.1 Ensure 'Boot-Start Driver Initialization Policy' is set to 'Enabled: Good, unknown and bad but critical'

Problem:

If your malware detection application does not include an Early Launch Antimalware boot-start driver or if your Early Launch Antimalware boot-start driver has been disabled, all boot-start drivers are initialized.

Improvement:

This policy setting allows you to specify which boot-start drivers are initialized based on a classification determined by an Early Launch Antimalware boot-start driver. The Early Launch Antimalware boot-start driver can return the following classifications for each boot-start driver:

Good: The driver has been signed and has not been tampered with.

Bad: The driver has been identified as malware. It is recommended that you do not allow known bad drivers to be initialized.

Bad, but required for boot: The driver has been identified as malware, but the computer cannot successfully boot without loading this driver. Unknown: This driver has not been attested to by your malware detection application and has not been classified by the Early Launch Antimalware boot-start driver.

If you enable this policy setting you will be able to choose which boot-start drivers to initialize the next time the computer is started.

This policy setting helps reduce the impact of malware that has already infected your system.

18.8.19.2 Ensure 'Configure registry policy processing: Do not apply during periodic background processing' is set to 'Enabled: FALSE'

Problem:

When background updates are disabled, policy changes will not take effect until the next user logon or system restart.

Improvement:

Setting this option to false (unchecked) will ensure that domain policy changes take effect more quickly, as compared to waiting until the next user logon or system restart.

18.8.19.3 Ensure 'Configure registry policy processing: Process even if the Group Policy objects have not changed' is set to 'Enabled: TRUE'

Problem:

if disabled, group policies are not reapplied if they have not been changed. Unauthorized local changes configured that do not match the domain-based Group policy settings goes undetected.

Improvement:

Setting this option to true (checked) will ensure unauthorized changes that might have been configured locally are forced to match the domain-based Group Policy settings again.

18.8.19.4 Ensure 'Continue experiences on this device' is set to 'Disabled'

Problem:

Windows device is allowed to participate in cross-device experiences. A cross-device experience is when a system can access app and send messages to other devices

If you enable this policy setting, the Windows device is discoverable by other Windows devices that belong to the same user, and can participate in cross-device experiences.

A cross-device experience is when a system can access app and send messages to other devices. In an enterprise environment only trusted systems should be communicating within the network. Access to any other system should be prohibited.

Improvement:

In an enterprise managed environment only trusted systems should be communicating within the network. Access to any other system should be prohibited.

18.8.25.1 Ensure 'Block user from showing account details on sign-in' is set to 'Enabled'

Problem:

An attacker with access to the console (for example, someone with physical access or someone who is able to connect to the server through Remote Desktop Services) could view the name of the last user who logged on to the server. The attacker could then try to guess the password, use a dictionary, or use a brute-force attack to try and log on.

Improvement:

This policy prevents the user from showing account details (email address or user name) on the sign-in screen.

18.8.25.2 Ensure 'Do not display network selection UI' is set to 'Enabled'

Problem:

An unauthorized user could disconnect the PC from the network or can connect the PC to other available networks without signing into Windows.

Improvement:

Not displaying network selection UI prevents anyone from interacting with available networks UI on the logon screen. The PC's network connectivity state cannot be changed without signing into Windows.

18.8.25.3 'Do not enumerate connected users on domain-joined computers' is set to 'Enabled'

Problem:

A malicious user could use this feature to gather account names of other users, that information could then be used in conjunction with other types of attacks such as guessing passwords or social engineering.

Improvement:

This policy setting prevents connected users from being enumerated on domain-joined computers.

18.8.25.4 Ensure 'Enumerate local users on domain-joined computers' is set to 'Disabled'

Problem:

A malicious user could use this feature to gather account names of other users, that information could then be used in conjunction with other types of attacks such as guessing passwords or social engineering. The value of this countermeasure is small because a user with domain credentials could gather the same account information using other methods.

Improvement:

This policy setting prevents local users from being enumerated on domain-joined computers.

18.8.25.5 Ensure 'Turn off app notifications on the lock screen' is set to 'Enabled'

Problem:

App notifications might display sensitive business or personal data.

Improvement:

App notifications turned off to prevent display of sensitive business or personal data.

18.8.25.6 Ensure 'Turn on convenience PIN sign-in' is set to 'Disabled'

Problem:

A PIN is created from a much smaller selection of characters than a password, so in most cases a PIN will be much less robust than a password.

Improvement:

When convenience pin sign-in disabled, password login is more secure.

18.8.26.1 Ensure 'Untrusted Font Blocking' is set to 'Enabled: Block untrusted fonts and log events'

Problem:

Attacks that may originate from untrusted or attacker-controlled font files.

Improvement:

Blocking untrusted fonts helps improve your network and employee protection against font-processing-related attacks. Blocking prevent both remote (web-based or email-based) and local Exchange Online Protection(EOP) attacks that can happen during the font file-parsing process.
